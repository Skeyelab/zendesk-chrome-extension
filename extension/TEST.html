Zendesk Sidebar Extension - Testing Guide

Installation Instructions

  1. Open Chrome and navigate to chrome://extensions/
  2. Enable "Developer mode" using the toggle in the top right corner
  3. Click "Load unpacked" button
  4. Select this directory: zendesk-chrome-extension
  5. The extension should now be installed and visible in your extensions list

Testing the Extension

Note: Since this is a test environment, you'll need to test on an actual Zendesk instance.

Option 1: Test on a real Zendesk instance

  1. Navigate to your Zendesk instance (e.g., yourcompany.zendesk.com)
  2. Log in and open any ticket
  3. The sidebar should appear automatically on the right side
  4. You should see:
    • Ticket ID
    • Requester Name
    • Requester Email

Option 2: Test with a Zendesk trial account

  1. Sign up for a free Zendesk trial at zendesk.com
  2. Create a test ticket
  3. Open the ticket to see the sidebar in action

Extension Features

  • Automatic Detection: The sidebar appears automatically when viewing a ticket
  • Settings: Click the extension icon → "Open Settings" to configure
  • Domain Filter: Optionally restrict to specific Zendesk instances
  • Close Button: Click × in the sidebar header to hide it
Future Enhancements: This extension is designed to be extensible. Future versions will include integrations with external API systems like Swell.is, providing agents with quick access to customer data from multiple sources.

Troubleshooting

  • Sidebar not appearing? Make sure you're on a Zendesk ticket page (URL should contain /agent/tickets/)
  • Information not loading? The page might still be loading. Wait a few seconds for the dynamic content to appear.
  • Extension not working? Check the settings to ensure it's enabled and the domain filter (if set) matches your Zendesk instance.
